# AI Fluency Certification

## Overview

Complete the AI 101 course to earn your **AI Fluent** certification. That's it - one certification, no tiers, no competition.

## What "AI Fluent" Means

You can:

- Use Claude Code effectively in your daily work
- Write prompts that get useful results
- Know when AI helps and when it doesn't
- Help teammates get started

## Requirements

### For All Roles

- [ ] **Complete Weeks 1-4 (Mandatory)** - Setup, Prompting, Planning, TDD fundamentals
- [ ] **Submit a capstone** showing AI-assisted work relevant to your role

### Additional by Role

| Role | Mandatory | Optional (Encouraged) |
|------|-----------|----------------------|
| Developer | Weeks 1-4 | Weeks 5-9 (Skills, Agents, Hooks, Plugins) |
| QA | Weeks 1-4 | QA-specific advanced content |
| PM | Weeks 1-4 | PM-specific advanced content |

## Timeline

| Track | Mandatory | Full Course |
|-------|-----------|-------------|
| Developer | 4 weeks | 9 weeks |
| QA | 4 weeks | 6-7 weeks |
| PM | 4 weeks | 6-7 weeks |

**Need more time?** That's fine. Let your trainer know. There's no penalty for going slower - the goal is competence, not speed.

## Capstone Examples

What does a "good" capstone look like? Here are examples by role:

### Developer Capstone Examples

- **Feature implementation:** Build a small HOA feature (violation notice generator, fee calculator) using TDD with Claude
- **Bug fix + tests:** Fix a real bug from backlog, add comprehensive tests, document the AI-assisted process
- **Refactoring:** Take legacy code, refactor with AI guidance, maintain or improve test coverage

### QA Capstone Examples

- **Test suite:** Generate comprehensive test cases for an existing feature, including edge cases
- **Coverage gap analysis:** Identify gaps in existing test coverage, write tests to fill them
- **Test data generator:** Create a utility that generates realistic test data for common scenarios

### PM Capstone Examples

- **Feature spec:** Write a complete PRD for an upcoming feature using AI assistance
- **User story set:** Create 5+ user stories with acceptance criteria for a feature
- **BDD scenarios:** Write Gherkin scenarios for a complex workflow

## Submission Process

2. **Commit your capstone** to your branch
3. **Create MR** titled "AI Fluency - [Your Name]"
4. **Notify trainer** in #ai-exchange

## FAQ

**Q: Is certification required?**
A: AI fluency is an expected skill for all team members. The certification confirms you've built that skill.

**Q: What if I don't pass the review?**
A: You'll get feedback on what to improve. Resubmit when ready - unlimited attempts.

**Q: Can I take longer than the typical timeline?**
A: Yes. DM your trainer. We care about competence, not speed.

**Q: What if I'm already experienced with AI tools?**
A: Great! You'll move through the content faster and can help teammates. The shared group experience still has value - everyone learns from each other.

**Q: Who reviews submissions?**
A: DX Team members.
