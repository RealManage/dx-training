# Developer Certification Track

## Overview

The developer track is the standard AI 101 course curriculum. It covers code review, TDD, debugging, refactoring, and building custom skills.

## Track Duration

**9 weeks typical**

| Phase | Weeks | Focus |
| ----- | ----- | ----- |
| Foundations | 0-4 | Setup, prompting, planning, TDD |
| Advanced | 5-8 | Commands, skills, agents, hooks, automation |
| Capstone | 9 | Demonstrate AI-assisted development |

## Week-by-Week Content

### Week 0: AI Foundations (Optional)

- What AI can and can't do
- How language models work
- Setting expectations

### Week 1: Setup & Orientation

- Install Claude Code
- Create your CLAUDE.md
- First conversations with Claude

### Week 2: Prompting Foundations

- Writing effective prompts
- Context and specificity
- Iterating on results

### Week 3: Tactical Planning & Code Review

- Plan mode for complex tasks
- AI-assisted code review
- Creating review artifacts

### Week 4: Test-Driven Development

- Red-Green-Refactor with AI
- Generating test cases
- Test coverage improvement

### Week 5: Commands & Basic Skills

- Using built-in commands
- Creating custom commands
- Understanding skills

### Week 6: Agents & Hooks

- Configuring agents
- Creating hooks for automation
- Workflow customization

### Week 7: Plugins

- Plugin architecture
- Creating team plugins
- Sharing across projects

### Week 8: Real-World Automation

- Headless CLI scripts, batch processing, JSON pipelines
- Parallel processing and multi-turn scripts
- Context management and anti-patterns (shared session)
- Full CLI flags reference

### Week 9: Capstone

Demonstrate AI fluency with a real project:

- Build a feature with AI assistance
- Include tests and review artifacts
- Document what worked and what didn't

## What You'll Be Able to Do

After completing this track:

- [ ] Write effective prompts for coding tasks
- [ ] Use AI for code review and catch issues early
- [ ] Practice TDD with AI assistance
- [ ] Create custom commands and skills
- [ ] Configure agents and hooks for your workflow
- [ ] Build plugins for team use
- [ ] Integrate AI into CI/CD pipelines
- [ ] Know when AI helps and when it doesn't

## Capstone Requirements

Submit a project demonstrating:

| Requirement | Description |
| ----------- | ----------- |
| Feature | Working code for a real feature or improvement |
| Tests | Test coverage following TDD principles |
| Review Artifact | AI-assisted code review documentation |
| Reflection | Notes on what worked, what didn't, and why |

## Differences from Other Tracks

| Topic | Developer Track | QA Track | PM Track |
| ----- | --------------- | -------- | -------- |
| Week 4 | TDD | Test case generation | User stories |
| Week 5 | Commands & Skills | Bug analysis | Specs & docs |
| Week 6 | Agents & Hooks | Test automation | BDD scenarios |
| Capstone | Code project | Test suite | Feature spec |

## Beyond Certification: QA Track

Since RealManage developers own testing, the **[QA track](qa-track.md)** is recommended after completing your developer certification:

- Test case generation (builds on TDD)
- Bug analysis & triage
- Test automation assistance

## FAQ

**Q: What if I already know some of this?**
A: Great! You'll move through it faster and can help teammates. The shared group experience still has value.

**Q: What languages does the course cover?**
A: Focus is on C#/.NET and Angular (RealManage stack), but concepts apply to any language.

**Q: What if I'm new to TDD?**
A: Week 4 covers TDD basics. AI assistance makes it easier to learn.

**Q: Is the capstone graded?**
A: Reviewed, not graded. We're confirming you got value from training, not assigning scores.
